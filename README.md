    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-project-3---tweeter" class="anchor" href="#project-3---tweeter" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Project 3 - <em>Tweeter</em></h1>

<p><strong>Tweeter</strong> is an android app that allows a user to view home and mentions timelines, view user profiles with user timelines, as well as compose and post a new tweet. The app utilizes <a href="https://dev.twitter.com/rest/public">Twitter REST API</a>.</p>

<p>Time spent: <strong>30</strong> hours spent in total</p>

<h2><a id="user-content-user-stories" class="anchor" href="#user-stories" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>User Stories</h2>

<p>The following <strong>required</strong> functionality is completed:</p>

<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can <strong>sign in to Twitter</strong> using OAuth login process</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can <strong>switch between Timeline and Mention views using tabs</strong>

<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User is displayed the username, name, and body for each tweet</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User is displayed the <a href="https://gist.github.com/nesquena/f786232f5ef72f6e10a7">relative timestamp</a> for each tweet "8m", "7h"</li>
</ul></li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can <strong>compose and post a new tweet</strong>

<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can click a "Compose" icon in the Action Bar on the top right</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can then enter a new tweet from a second activity and then post this to twitter</li>
</ul></li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can navigate to <strong>view their own profile</strong>

<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can see picture, tagline, # of followers, # of following, and tweets on their profile.</li>
</ul></li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can <strong>click on the profile image</strong> in any tweet to see <strong>another user's</strong> profile.

<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can see picture, tagline, # of followers, # of following, and tweets of clicked user.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> Profile view includes that user's timeline of recent tweets</li>
</ul></li>
</ul>

<p>The following <strong>optional</strong> features are implemented:</p>

<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> While composing a tweet, user can see a character counter with characters remaining for tweet out of 140</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can <strong>pull down to refresh tweets</strong> in either timeline.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> User can <strong>search for tweets matching a particular query</strong> and see results.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> Improve the user interface and theme the app to feel twitter branded with colors and styles</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> When a network request is sent, user sees an <a href="http://guides.codepath.com/android/Handling-ProgressBars#progress-within-actionbar">indeterminate progress indicator</a></li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> User can <strong>"reply" to any tweet on their home timeline</strong>

<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> The user that wrote the original tweet is automatically "@" replied in compose</li>
</ul></li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> User can click on a tweet to be <strong>taken to a "detail view"</strong> of that tweet

<ul class="contains-task-list">
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> User can take favorite (and unfavorite) or retweet actions on a tweet</li>
</ul></li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> User can see embedded image media within the tweet item in list or detail view.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> Compose activity is replaced with a modal compose overlay.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" checked="" disabled=""> User can <strong>click a link within a tweet body</strong> on tweet details view. The click will launch the web browser with relevant page opened.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> Used Parcelable instead of Serializable leveraging the popular <a href="http://guides.codepath.com/android/Using-Parceler">Parceler library</a> when passing data between activities.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> Replaced all icon drawables and other static image assets with <a href="http://guides.codepath.com/android/Drawables#vector-drawables">vector drawables</a> where appropriate.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> User can view following / followers list through the profile of a user</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> Apply the popular Butterknife annotation library to reduce view boilerplate.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> Implement collapse scrolling effects on the Twitter profile view using <code>CoordinatorLayout</code>.</li>
<li class="task-list-item"><input type="checkbox" class="task-list-item-checkbox" disabled=""> User can <strong>open the twitter app offline and see last loaded tweets</strong>. Persisted in SQLite tweets are refreshed on every application launch. While "live data" is displayed when app can get it from Twitter API, it is also saved for use in an offline mode.</li>
</ul>
